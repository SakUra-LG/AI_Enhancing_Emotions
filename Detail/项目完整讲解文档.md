## 项目概述

### 这是什么项目？
这是一个**AI辅助小说创作系统**，专门用于生成仙侠小说的细节内容。它能够：
- 根据你的简单描述，自动生成详细的武打场景
- 改写中国神话故事，加入现代幽默元素
- 生成其他仙侠小说情节片段

### 项目名称
**AI_Novel_Detail1** - AI小说细节生成系统

### 核心特点
- 🤖 使用AI大语言模型（通义千问）生成内容
- 🔍 使用RAG技术（检索增强生成）提高内容质量
- 📚 内置知识库，包含大量小说素材和专业知识
- ⚡ 支持GPU加速，处理速度快

## 项目用途

### 主要功能

#### 1. **武打场景生成**
- **输入**：你描述一个武打场景的需求（如"两位高手在古寺对决"）
- **输出**：生成一段1500字左右的详细武打场景
- **特点**：严格按照"压制→绝境→转机→反杀"的结构生成，情节跌宕起伏

#### 2. **神话故事改写**
- **输入**：你要求改写某个神话（如"改写盘古开天地的故事"）
- **输出**：生成一个幽默风格的神话改写版本
- **特点**：保留核心情节，加入现代幽默元素，但避免脏话和过度现代化

#### 3. **通用小说情节生成**
- **输入**：你描述任何小说情节需求
- **输出**：生成符合要求的仙侠小说片段
- **特点**：自动检索相关素材，结合专业知识生成

### 使用场景
- 小说作者需要快速生成武打场景细节
- 需要改写传统神话故事，加入现代元素
- 创作灵感不足时，获取AI生成的参考内容

### 核心技术：RAG（检索增强生成）

**RAG是什么？**
- **R**etrieval（检索）：从知识库中找到相关内容
- **A**ugmented（增强）：用检索到的内容增强提示词
- **G**eneration（生成）：用增强后的提示词生成内容

**为什么需要RAG？**
- 大语言模型虽然强大，但可能不知道最新的素材或专业知识
- RAG让AI在生成前先"查阅资料"，生成的内容更准确、更专业

## 工作流程详解

### 阶段一：知识库初始化（首次使用或更新素材时）

#### 步骤1：处理小说素材库
```bash
python Handle_Content.py
```

**这个脚本做了什么？**
1. 读取 `knowledgeBase/Content.txt`（包含大量小说片段）
2. 解析文本，提取每个主题下的段落
3. 使用BGE模型将每个段落转换为向量（1024维数字）
4. 保存向量到 `features_Theme.npy`
5. 保存结构化内容到 `themes_Content.json`

**为什么需要向量化？**
- 计算机不能直接理解文字
- 向量是数字，可以计算相似度
- 相似的文本会有相似的向量

#### 步骤2：处理专业知识库
```bash
python Handle_Profession.py
```

**这个脚本做了什么？**
1. 读取 `knowledgeBase/Professional.txt`（包含专业知识）
2. 解析"名称：描述"格式的知识点
3. 向量化每个知识点
4. 保存到 `features_profession.npy` 和 `features_profession.json`

### 阶段二：运行主程序（日常使用）

#### 步骤1：启动程序
```bash
python main.py
```

#### 步骤2：输入提示词
程序会提示：`请输入你的问题:`

**示例输入：**
```
请你写一段武打情节，主角是一位年轻剑客，对手是一位经验丰富的拳法宗师，在古寺大殿中生死对决
```

#### 步骤3：系统自动处理（你不需要手动操作）

**3.1 意图识别**
- 检查输入中是否包含关键词
- 如果是"武打/战斗/对决" → 调用武打场景生成函数
- 如果是"神话/盘古/后羿" → 调用神话改写函数
- 其他 → 调用通用生成函数

**3.2 语义检索**
- 调用 `Search_content.py`：
  - 将你的输入转换为向量
  - 在知识库中查找最相似的5个段落
  - 随机选择1个作为参考素材
- 调用 `Search_profession.py`：
  - 查找最相关的专业知识（如"情节设计技巧"）

**3.3 构建提示词**
系统会构建一个包含以下内容的提示词：
```
角色：你是一个仙侠小说作者...
参考内容：[检索到的小说素材]
专业知识：[检索到的专业知识]
要求：[你的输入]
```

**3.4 调用AI生成**
- 调用通义千问API
- 传入构建好的提示词
- 等待AI生成内容（通常几秒到几十秒）

**3.5 后处理**
- 清理Markdown格式符号
- 去除多余的注释和说明
- 格式化输出

**3.6 显示结果**
- 在控制台打印生成的内容
- 如果是武打场景，会保存到文件

## 核心文件说明

### 1. 配置文件

#### `config.py`
```python
API_Key_QW = "sk-..."  # 通义千问API密钥
MAX_TOKENS = 8192      # 最大token数
```
**作用**：存储API密钥和配置参数

### 2. 知识库处理文件

#### `Handle_Content.py`
**功能**：处理小说素材库，生成向量
**输入**：`knowledgeBase/Content.txt`
**输出**：
- `knowledgeBase/features_Theme.npy`（向量文件）
- `knowledgeBase/themes_Content.json`（结构化内容）

**关键代码逻辑**：
1. 读取文本文件
2. 解析主题和段落（使用正则表达式）
3. 批量向量化（使用BGE模型）
4. 保存结果

#### `Handle_Profession.py`
**功能**：处理专业知识库，生成向量
**输入**：`knowledgeBase/Professional.txt`
**输出**：
- `knowledgeBase/features_profession.npy`
- `knowledgeBase/features_profession.json`

### 3. 检索模块

#### `Search_content.py`
**功能**：根据用户输入检索相关小说素材
**工作流程**：
1. 加载知识库（向量+内容）
2. 将用户输入向量化
3. 计算余弦相似度
4. 找到最相似的5个段落
5. 随机选择1个返回

**关键函数**：
- `vectorize_text(text)`：将文本转为向量
- `find_most_similar()`：找最相似的内容
- `searchresult_content(user_input)`：主函数

#### `Search_profession.py`
**功能**：检索相关专业知识
**工作流程**：类似Search_content.py，但返回最匹配的专业知识

### 4. 主程序

#### `main.py`
**功能**：整合所有模块，生成最终内容
**核心函数**：

1. **`call_qianwen_api()`**
   - 调用通义千问API
   - 包含重试机制（最多5次）
   - 处理各种错误

2. **`chat_once(prompt)`**
   - 单轮对话函数
   - 调用检索模块获取参考内容
   - 构建系统提示词
   - 调用API生成内容

3. **`generate_fight_scene_with_reversal(prompt)`**
   - 专门生成武打场景
   - 使用特殊的模板，强制按"压制→转机→反杀"结构生成

4. **`generate_myth_rewrite(prompt)`**
   - 专门改写神话故事
   - 包含详细的风格约束（避免脏话、过度现代化等）

5. **`clean_markdown(text)`**
   - 清理Markdown格式
   - 处理转义字符
   - 删除注释和多余空行

**主函数逻辑**：
```python
if __name__ == "__main__":
    prompt = input("请输入你的问题: ")
    
    # 判断意图
    if "神话" in prompt:
        answer = generate_myth_rewrite(prompt)
    elif "武打" in prompt:
        answer = generate_fight_scene_with_reversal(prompt)
    else:
        answer = chat_once(prompt)
    
    print(answer)
```

#### `demo.py`
**功能**：演示脚本，自动生成一个武打场景示例
**用途**：快速测试系统是否正常工作

### 5. 知识库文件

#### `knowledgeBase/Content.txt`
**格式**：
```
1.玄幻斗法

"金光乍现，千百道剑气纵横..."

2.武功对决
```

**作用**：存储专业知识（地名、修炼、药物等），帮助生成更专业的内容

### 6. 模型文件

#### `bge_large_zh/` 目录
包含BGE-Large-ZH模型的所有文件：

## 技术栈详解

### 1. Python（编程语言）
- **版本**：建议Python 3.8+
- **作用**：整个项目用Python编写
- **学习重点**：
  - 基础语法（变量、函数、类）
  - 文件操作（读写txt、json）
  - 正则表达式（解析文本）
  - 异常处理（try-except）

### 2. PyTorch（深度学习框架）
- **作用**：加载和运行BGE模型
- **关键概念**：
  - **Tensor**：多维数组，类似NumPy数组但支持GPU
  - **模型加载**：`AutoModel.from_pretrained()`
  - **推理模式**：`torch.inference_mode()`（不计算梯度，更快）
  - **设备选择**：`device = "cuda" if torch.cuda.is_available() else "cpu"`

**示例代码理解**：
```python
# 加载模型到GPU或CPU
model = AutoModel.from_pretrained(model_path).to(device)

# 推理时不需要计算梯度（更快）
with torch.inference_mode():
    outputs = model(**inputs)
```

### 3. Transformers（Hugging Face库）
- **作用**：提供预训练模型的接口
- **关键类**：
  - `AutoTokenizer`：自动加载分词器
  - `AutoModel`：自动加载模型
- **学习重点**：
  - 如何加载预训练模型
  - 如何使用tokenizer处理文本
  - 如何获取模型输出

### 4. BGE-Large-ZH（中文嵌入模型）
- **全称**：BAAI General Embedding Large Chinese
- **作用**：将中文文本转换为1024维向量
- **特点**：
  - 专门针对中文优化
  - 理解中文语义和文化背景
  - 向量质量高，检索精度好

**工作原理**：
```
文本："两位高手在古寺对决"
    ↓ (BGE模型处理)
向量：[0.123, -0.456, 0.789, ..., 0.234] (1024个数字)
```

### 5. NumPy（数值计算库）
- **作用**：处理向量和数组计算
- **关键操作**：
  - `np.vstack()`：垂直堆叠数组
  - `np.argsort()`：排序并返回索引
  - `np.load()` / `np.save()`：保存和加载数组

### 6. scikit-learn（机器学习库）
- **作用**：计算余弦相似度
- **关键函数**：
  ```python
  from sklearn.metrics.pairwise import cosine_similarity
  similarities = cosine_similarity(query_vector, vectors)
  ```
- **余弦相似度**：衡量两个向量相似程度的指标（0-1之间，1表示完全相同）

### 7. dashscope（阿里云SDK）
- **作用**：调用通义千问API
- **关键代码**：
  ```python

  dashscope.api_key = API_Key_QW

  ```

### 8. 正则表达式（re模块）
- **作用**：解析文本文件
- **示例**：
  ```python
  # 匹配"名称：描述"格式
  pattern = r'(.+?)：(.+?)(?=\n.+?：|$)'
  matches = re.findall(pattern, text)
  ```

### 9. JSON（数据格式）
- **作用**：存储结构化数据（知识库元数据）
- **操作**：
  ```python
  # 读取
  with open('file.json', 'r', encoding='utf-8') as f:
      data = json.load(f)
  
  # 写入
  with open('file.json', 'w', encoding='utf-8') as f:
      json.dump(data, f, ensure_ascii=False, indent=2)
  ```

```python
top_indices = np.argsort(similarities)[::-1][:top_k]
```

## 总结

这个项目是一个**RAG（检索增强生成）系统**，核心流程是：

1. **预处理**：将知识库文本转换为向量（Handle_Content.py, Handle_Profession.py）
2. **检索**：根据用户输入找到相关内容（Search_content.py, Search_profession.py）
3. **生成**：用检索结果增强提示词，调用AI生成内容（main.py）

**核心技术**：
- BGE模型：文本向量化
- 余弦相似度：检索相似内容
- 通义千问API：生成最终内容

**学习重点**：
1. Python编程基础（必须）
2. PyTorch和Transformers（核心）
3. RAG技术理解（关键）
4. API调用（实用）
