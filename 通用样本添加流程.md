# 通用样本添加流程（`universal_samples.txt`）

本文档说明：如何在项目中**添加新的通用样本**并让它参与后续的向量检索 / RAG 生成。

---

## 你要改哪个文件？

- **主入口文件**：`bert_excitation_train/data/universal_samples.txt`
  - `scripts/handle_universal_samples.py` 会**只读取**这个文件来解析样本并重建向量库。
- 下面这些文件是**脚本生成物**，一般不要手改：
  - `bert_excitation_train/data/universal_samples_data.json`（结构化样本数据）
  - `bert_excitation_train/data/universal_samples_vectors.npy`（样本向量）

> 说明：仓库里可能还有诸如 `universal_samples_scored.txt` 之类的文件，它们通常是“展示/参考/打分结果”，不一定会被主流程读取。**是否生效以 `universal_samples.txt` 为准。**

---

## 第 1 步：在 `universal_samples.txt` 末尾追加一个样本块

每个样本以 `## ` 标题行开始，后面跟一组固定字段。示例（直接复制改内容即可）：

```text
## 悬疑推理-4
**情绪标签**: 紧张, 不安, 好奇
**场景标签**: 现代都市, 夜晚, 室内
**冲突标签**: 搜索真相, 危机逼近
**动作标签**: 观察, 思考, 推理
**情节标签**: 发现异常, 伏笔铺垫
**评分**: 80.00
**内容**: 这里写你的高质量片段正文……
```

### 格式要求（很重要）

- **标题行**：必须以 `## ` 开头
  - 标题文本会作为该条样本的 `title`，并同时作为 `category`（脚本里是“标题即类别”）。
- **字段名**：必须写成下面这些固定前缀之一（与已有样本完全一致）：
  - `**情绪标签**:`
  - `**场景标签**:`
  - `**冲突标签**:`
  - `**动作标签**:`
  - `**情节标签**:`
  - `**评分**:`
  - `**内容**:`
- **标签分隔符**：同一行多个标签用英文逗号 `,` 分隔（脚本会对每个标签做 `strip()` 去空格）。
- **内容不能为空**：脚本只会收录带 `**内容**:` 且内容非空的样本。

## 第 2 步：运行脚本重建样本库（必须）

在 Windows PowerShell 里进入项目根目录（包含 `bert_excitation_train/` 的那个目录）：

```powershell
cd "D:\Study\College\Scientific research\张颖——AI小说自动生成\张颖——AI小说自动生成\bert_excitation_train\AI_Novle"
```

运行：

```powershell
python .\bert_excitation_train\scripts\handle_universal_samples.py
```

脚本会：

- 读取 `bert_excitation_train/data/universal_samples.txt`
- 解析样本并输出解析数量
- 生成/覆盖：
  - `bert_excitation_train/data/universal_samples_data.json`
  - `bert_excitation_train/data/universal_samples_vectors.npy`

---

## 第 3 步：确认是否生效（可选但推荐）

### 方式 A：在 JSON 中搜索标题

打开 `bert_excitation_train/data/universal_samples_data.json`，搜索你新样本的标题（如 `悬疑推理-4`），能找到则说明解析成功。

### 方式 B：运行依赖通用样本库的脚本

如果你运行下面这些脚本时不再提示“请先运行 handle_universal_samples.py”，且能检索到新样本，说明向量库已更新：

- `bert_excitation_train/scripts/smart_sample_search.py`
- `bert_excitation_train/scripts/enhanced_rag_generator.py`
- `bert_excitation_train/scripts/universal_generator.py`

## 常见问题（Troubleshooting）

### 1) 运行脚本时提示模型路径相关错误

`bert_excitation_train/scripts/handle_universal_samples.py` 内部使用了一个本机模型目录变量 `model_path`。

- 如果你的机器上模型目录不同，请把 `model_path` 改成你本地的 `bge_large_zh` 目录路径（示例：项目根目录下的 `bge_large_zh/`）。

### 2) 新样本没被收录

重点检查：

- 是否真的写进了 `bert_excitation_train/data/universal_samples.txt`
- 标题行是否以 `## ` 开头
- 是否包含一行 `**内容**:` 且内容非空
- 标签行前缀是否完全一致（例如必须是 `**情节标签**:`，不能写成别的）
